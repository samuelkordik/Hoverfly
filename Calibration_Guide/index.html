Hoverfly — Calibration & Tuning Guide

A repeatable, ordered tune-up for the Ender 3 Pro · SKR Mini E3 V2.0 · Sprite Extruder Pro (direct drive) · CRTouch · Elegoo PLA

How to use this guide. Work the steps in order — each assumes the previous passed. Print the linked STL, compare your result to the reference diagrams and pass criteria, then adjust and re-test. Want a guided session? Open this folder with Claude — see CLAUDE.md — and it'll walk you through step by step. The same procedure in plain-text/machine-readable form is GUIDE.md.

Note: the diagrams below are schematic SVGs (the automated photo download was interrupted by a usage limit). Real reference photos can be added later.
Before you start
  • Filament: Elegoo PLA — hotend ≈190–220 °C, bed ≈50–60 °C.
  • Save any stored value with M500. M501 reloads, M502 resets to firmware defaults.
  • Change one thing at a time, then re-test.
  • Pair this with the firmware review and Orca guide.
No G-code terminal? Hoverfly has no Pi / OctoPrint / USB console, so each step's Run line says how to do it without one:
  • LCD menu — steps/mm, acceleration, babystep Z, Store Settings (M500), mesh viewer.
  • Orca → Calibration — generates the temperature / flow / pressure-advance / retraction tests as prints.
  • SD macros in macros/ — copy to the SD card once, then "Print" the named file (SET_* files are edit-first).
The <pre> blocks show the raw M-codes for reference — you don't type them. M48 / M503 / M420 V are serial-only (use the LCD mesh viewer; defer M48 to the planned Klipper console).
First-run order (fresh printer): the steps are written 1→9, but Step 4 (Z-offset) needs an active mesh — a good first layer is mesh-compensated. On a brand-new printer run 1 → 2 → 3 → 9 (build first mesh) → 4 → 5–8. PID and E-steps (Steps 2–3) are independent and can precede the mesh; the mesh itself (Step 9) builds fine before Z-offset is dialed (it's relative). After that initial pass the numeric 1→9 order is fine for re-tuning.

The 9 steps

1 · Bed tramming (mechanical level) 2 · Hotend PID autotune 3 · E-steps & flow 4 · Z-offset & first layer 5 · Temperature tower 6 · Retraction & pressure/linear advance 7 · Speed & acceleration 8 · Ringing & input shaping 9 · Bed mesh (ABL) & validation

1 · Bed tramming (mechanical level)

No STLLive moves

Goal: get all four corners within ~0.1 mm mechanically, so the CRTouch mesh only has to correct small variation — not a skewed bed.

Do it: heat the bed to 60 °C (it changes shape when hot), home, drop the nozzle to ~0.2 mm, and use the paper-drag test at each corner knob and the center.

M140 S60      ; heat bed
G28           ; home
G1 Z0.2 F600  ; nozzle near bed; jog to each corner and adjust knobs
; M211 S0    ; (reference only — disables soft endstops; re-enable with M211 S1 after)
The supported flow here is LCD jog only — no macro sends M211, so soft endstops stay on. If you ever disable them manually (M211 S0), re-enable with M211 S1 afterward (they also reset on power-cycle).
Pass: corner-to-corner nozzle height varies < 0.1 mm. If fail: adjust knobs; if a corner can't reach, check springs and that both Z leadscrews sit at equal height (power off, hand-turn to match).

2 · Hotend PID autotune

No STLSD macro

Goal: hold the hotend within ±0.5 °C. Tune with the part fan in its printing state.

Run: Print macros/PID_HOTEND.gcode from SD — it autotunes with the fan on, applies, and saves. M303 pauses for an LCD acknowledge at the end. The stored constants are generic stock values, so a fresh autotune for the Sprite block is worthwhile.

M106 S255            ; part fan on (print-like cooling)
M303 E0 S205 C8 U1   ; autotune hotend @205C, 8 cycles, apply
M107                 ; fan off
M500                 ; save
Pass: steady temperature, no sawtooth > ±1 °C, no runaway trip. If fail: re-run with C10; check hotend fan and thermistor/heater seating. Existing values (21.73 / 1.54 / 76.55) are a fine fallback.

3 · E-steps & flow

single_wall_flow_25mm.stl

E-steps are already 423 (calibrated for the Sprite). Verify, then dial slicer flow.

Run: E-steps — Print macros/ESTEPS_TEST.gcode, measure, then set via the LCD Steps/mm menu or macros/SET_ESTEPS.gcode (edit first). Flow — use Orca's Flow Rate calibration.

3a · Verify E-steps

M83 ; G1 E100 F60     ; heat to 205C first; mark 120mm above extruder, extrude 100mm
; if actual ≠ 100mm:  M92 E<423 * 100 / actual>  then  M500
Measure the remaining filament 2–3 times (or re-run) and average — a 1 mm read error is ~1% of E-steps. Mark and measure against a fixed point on the extruder body, not the nozzle tip. The test extrudes 100 mm and does not retract, so expect a ~100 mm strand hanging from the nozzle — snip it off afterward. Confirm on LCD: Configuration → Advanced Settings → Steps/mm → E steps/mm.

3b · Flow / extrusion multiplier

Slice the flow cube with 1 perimeter, 0 top/bottom, 0% infill (or Vase mode), 0.2 mm layer. Measure the wall with calipers.

Pass: wall within ±0.02–0.03 mm of target. If fail: new_flow = old_flow × target_wall / measured_wall (Orca → Filament → Flow ratio), reprint.

4 · Z-offset & first layer

firstlayer_patch_60mm.stl

Print the patch; adjust live with babystepping (double-click encoder → Babystep Z).

Run: Until you reflash with BABYSTEP_ZPROBE_OFFSET (firmware review §3), use macros/SET_ZOFFSET.gcode (edit M851 Z<value> + M500) — that persists. The LCD live-babystep method (double-click encoder → Babystep Z, dial by feel) is great for finding the value, but with BABYSTEP_ZPROBE_OFFSET off the babystep is not folded into the probe offset by Store Settings and is lost on reboot; once that flag is enabled, the live method persists with Configuration → Store Settings.

G28
M851 Z-3.49     ; current probe Z-offset (info)
; during layer 1: babystep DOWN if too high, UP if scraping
M500            ; save (persists Z only if BABYSTEP_ZPROBE_OFFSET is enabled — see firmware review §3)
Confirm on LCD: Configuration → Probe Z Offset.
Pass: solid patch, no gaps, no elephant-foot ooze, peels in one piece. If fail: adjust ±0.02–0.05 mm and reprint.

5 · Temperature tower

temp_tower_70mm.stlor a labeled tower — see ATTRIBUTION.md

Sweep ~220 → 190 °C (Orca's temperature-tower calibration inserts the per-band M104). Judge each band for adhesion, bridging, overhangs, and stringing.

Run: Orca → Calibration → Temperature tower generates the whole print — no codes.

Pass: a band that snaps strong with clean overhangs/minimal strings. If fail: all bands string → too hot / fix retraction (Step 6); all delaminate → too cold or over-cooled.
Loop back if needed: Step 2 autotuned PID at 205 °C. If your chosen tower temp differs from 205 by more than ~10 °C, re-run macros/PID_HOTEND.gcode after editing it to autotune at your final temp (M303 E0 S<temp>), then M500 — PID constants are mildly temperature-dependent, so tuning near your real print temp is best. A ±5–10 °C delta rarely matters.

6 · Retraction & pressure / linear advance

retraction_test_50mm.stlOrca PA test

Tune pressure advance first, then retraction — good PA relieves nozzle pressure and shrinks the retraction you need. Direct drive = short retraction: start ~0.6–0.8 mm, tune distance down until stringing returns, then back up one notch.

Run: Orca → Calibration → Pressure Advance (Pattern, Direct Drive) and Retraction test both generate prints. Apply PA via Orca, or macros/SET_PA.gcode; set retraction in Orca's filament/printer settings.

Retraction speed is capped at 25 mm/s — the firmware E max feedrate. Anything higher (the 35–45 mm/s you'll see quoted for Bowden) is silently clamped, so tune distance, not speed (or raise DEFAULT_MAX_FEEDRATE E first).
M83
; Orca Pressure Advance (Pattern, Direct Drive): sweep K 0.0–0.10 step 0.005,
; then a fine second pass at 0.002 around the best band. Or set Marlin K directly:
M900 K0.04     ; Linear Advance K — small values for Sprite direct drive
M500
; pick ONE: Orca PA or Marlin K, not both
Pass: no/wispy strings; corners crisp with no bulge and no gaps at perimeter starts. If fail: more retraction distance for strings (speed is capped at 25 mm/s; ≤~1.5 mm on direct drive); K up if corners bulge, down if they show gaps.
Note: Orca's Enable pressure advance checkbox injects M900 K into the print G-code and overrides firmware K every print, so untick it when you're relying on SET_PA.gcode/M500 (Marlin K).

7 · Speed & acceleration

calibration_cube_20mm.stl

Raise acceleration in steps and watch for ghosting and skipped steps. Firmware default print accel is a conservative 500; the light Sprite can usually go higher.

Run: set accel via the LCD Acceleration menu or macros/SET_ACCEL.gcode (edit values first), then print the cube from SD to test.

M201 X1000 Y1000   ; raise max accel to test
M204 P1000         ; print accel
M500               ; save only the values you settle on
Confirm on LCD: Configuration → Advanced Settings → Acceleration.
Pass: 20 mm cube measures 20 ±0.2 mm on each axis; no layer shift; acceptable ringing. If fail: back off accel; if dimensions are consistently off, check belt tension and steps/mm (X/Y 80, Z 400).

8 · Ringing & input shaping

ringing_tower_60mm.stl

Ghosting (echoes after sharp corners) comes from frame resonance. A ringing tower's oscillation spacing encodes the frequency.

Run: only if Input Shaping is compiled and fits. Sweep with the slicer after-layer macro (below), then apply with macros/SET_SHAPER.gcode (edit the measured frequencies first).

First confirm it fits the F103. Enable INPUT_SHAPING_X/Y + SHAPING_MIN_FREQ 20.0 (bounds the SRAM step buffer), build, and check the linker doesn't say "region RAM overflowed" before printing. Then print the tower at high accel (M204 S1500) sweeping frequency per layer:

; slicer "after layer change" G-code — sweeps M593 F 15→60 Hz up the tower:
M593 F{(layer_num < 2 ? 0 : 15 + 45.0 * (layer_num - 2) / 297)}

; read the smoothest band, convert height→Hz, then set per axis:
M593 X F40 ; M593 Y F38             ; frequencies (YOUR measured values)
M593 X D0.15 ; M593 Y D0.15         ; damping (zeta) — refine in 0.05 steps
M500
Then re-tune Linear Advance K (Step 6) after Input Shaping — the sweep runs with LA off, so K is finalized last. If the firmware won't build/flash with IS at all, instead lower acceleration (Step 7) to the ring-free value. First-class resonance compensation is part of the planned Klipper migration.

9 · Bed mesh (ABL) & validation

firstlayer_patch_60mm.stl

Probe the bilinear mesh at print temperature, store it, and validate the first layer everywhere.

Run: Print macros/BUILD_MESH.gcode from SD (heats, homes, probes, saves, enables). View the result on the LCD mesh viewer — M420 V and M48 are serial-only, so skip M48 probe-repeatability until the Klipper console.

Probe at real print temp. PREHEAT_BEFORE_LEVELING is on, but LEVELING_BED_TEMP is currently set to 50 (Configuration.h:1936) — below your ~60 °C PLA bed, so the mesh is probed ~10 °C cold. An aluminum bed warps differently when hot, so a mesh probed cold misses warp that's present while printing. Set #define LEVELING_BED_TEMP 60 (reflash), or probe fresh in start G-code after the bed soaks: M190 S60 → G4 S300 (soak) → G28 → G29 → M500.
Confirm on LCD: Configuration → (Bed) Leveling → Mesh viewer.
M140 S60 ; M104 S150   ; bed to print temp; warm nozzle (low ooze)
M190 S60 ; M109 S150    ; wait for both
G28
G29                    ; probe bilinear mesh
M500                   ; store
M420 S1 Z10            ; enable leveling + fade height (put M420 S1 in your slicer start G-code)
; optional, separately: G26 B60 H205  (if G26_MESH_VALIDATION compiled) prints a mesh-check pattern
Pass: mesh Z range < ~0.3–0.4 mm (view on the LCD mesh viewer); first layer equally good center and edges. If fail: redo Step 1 (tramming); consider 5×5 grid and a smaller PROBING_MARGIN (firmware review §5–6).
Save order when done: PID saved → flow set in Orca → Z-offset saved → best temp + retraction + K saved → accel set → mesh stored and M420 S1 in slicer start G-code. Each macro saves with M500; or use the LCD → Store Settings after an LCD change.
Flash-emulated EEPROM caveat (SKR Mini E3 V2.0): there's no real EEPROM — M500 writes to a reserved flash sector and is reliable day-to-day, but a reflash can change the EEPROM layout and reset all stored values to defaults. After any firmware update, re-verify (and re-save) PID, E-steps, Z-offset, K, accel, and rebuild the mesh — don't assume they survived.
